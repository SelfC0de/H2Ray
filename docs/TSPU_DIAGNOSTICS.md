# Диагностика сетевых ограничений

## Ограничение результата

Приложение не может достоверно определить наличие или модель ТСПУ у провайдера.
Оно может только сравнивать наблюдаемое сетевое поведение прямого и
проксированного соединения. Ошибки маршрутизации, DNS, сервера и обычный packet
loss могут давать те же симптомы.

## Безопасные проверки

Планируемая диагностика должна выполнять независимые тесты:

1. доступность DNS через системный resolver и через настроенный Xray DNS;
2. TCP connect к нескольким контрольным адресам на 80 и 443;
3. TLS handshake с корректным SNI и сравнение с подключением по IP;
4. HTTP-запрос через direct и proxy outbound;
5. проверку UDP/443 отдельно от TCP/443;
6. фиксацию timeout, reset, TLS alert, HTTP status и времени этапов.

Результат должен называться «признаки фильтрации», а не «ТСПУ обнаружено».

## Механизмы Xray

- REALITY с RAW/XHTTP/gRPC уменьшает отличимость TLS-трафика.
- XTLS Vision применим к VLESS поверх TCP+TLS/REALITY.
- XHTTP и FinalMask требуют согласованной конфигурации с сервером.
- Freedom `fragment` может дробить ClientHello, но относится к Freedom outbound
  и не должен автоматически добавляться к произвольному VLESS-профилю.
- UDP `noises` способен нарушить обычные соединения и не должен включаться по
  умолчанию.
- Блокировка QUIC заставляет приложения использовать TCP, но может ухудшить
  работу приложений, которые не умеют откатываться с QUIC.

## Политика H2Ray

H2Ray не изменяет импортированный transport без необходимости совместимости с
текущим Xray-core. Экспериментальные маскировки должны добавляться отдельным
профилем, показывать итоговый JSON и требовать явного включения пользователем.
