# Выпуск и обновление H2Ray

Workflow `Build Android APK` запускается вручную. Он принимает публичный
четырёхкомпонентный `version_name`, собирает только release-вариант, подписывает
его постоянным ключом и публикует `H2Ray.apk` в artifact и GitHub Release.

Приложение проверяет endpoint последнего GitHub Release при запуске. Если его
версия выше установленной и в нём есть asset с точным именем `H2Ray.apk`, на
главном экране появляется уведомление об обязательном обновлении. Кнопка в
настройках запускает ту же проверку вручную.

Начиная с `1.0.0.2`, APK скачивается системным Android DownloadManager внутри
приложения. DownloadManager сохраняет загрузку при медленной или временно
пропавшей сети. Перед передачей APK системному установщику H2Ray останавливает
VPN. Если VPN был активен, приложение переподключается при первом запуске уже
установленной версии. Android требует от пользователя отдельно разрешить
установку из этого источника и подтвердить установку APK.
Перед открытием установщика приложение сверяет имя пакета, версию и сертификат
скачанного APK с уже установленным H2Ray.

Начиная с `1.0.0.3`, меню профилей импортирует share-ссылки из `.txt`, клиентские
Xray-конфигурации из `.json` и ссылки из QR-кодов. Серверный `inbound` нельзя
преобразовать в подключение без внешнего адреса сервера и клиентского
UUID/пароля; приложение выводит для такого файла диагностическую ошибку.

`versionCode` равен монотонно растущему номеру запуска workflow. Поэтому новая
сборка устанавливается поверх старой, если соблюдены все условия:

- `applicationId` остаётся `com.h2ray.app`;
- используется тот же release-ключ;
- номер запуска новой сборки больше номера предыдущей.

Release-ключ и пароли хранятся в GitHub Actions Secrets:

- `H2RAY_KEYSTORE_BASE64`;
- `H2RAY_KEYSTORE_PASSWORD`;
- `H2RAY_KEY_ALIAS`;
- `H2RAY_KEY_PASSWORD`.

Локальный каталог `signing/` исключён из Git. Его резервную копию необходимо
хранить отдельно. Утрата ключа делает обновление уже установленных приложений
невозможным.

Существующие debug-сборки подписаны другим ключом. Для перехода с debug на
первую release-сборку потребуется однократно удалить старое приложение.
Последующие release-сборки устанавливаются поверх него без удаления.

## 1.0.0.14

- Добавлены параллельные DNS-запросы Xray.
- Добавлен отключённый по умолчанию экспертный режим FakeDNS.
- Добавлены Happy Eyeballs RFC 8305 и TCP user timeout.
- Импортированные TLS pinning, ECH и curve-параметры больше не удаляются.
- REALITY-нормализация сохраняет поддерживаемые ядром клиентские поля.
- В диагностический отчёт добавлено состояние новых сетевых параметров.

Stats, Metrics и Observatory не заявлены как реализованные: используемый
`libXray.aar` не экспортирует необходимые API. Для них требуется изменение и
пересборка Go-обёртки, после чего Android `TrafficStats` можно заменить
статистикой конкретного outbound.

## 1.0.0.15

- Исправлено отображение режима выбора приложений «Через VPN»/«Вне VPN».
- Добавлено явное описание действия флажков и счётчик выбранных приложений.
- Исправлено переключение режима и удаление устаревших пакетов из правил.
- H2Ray всегда включается в allowed-list режима «Через VPN», чтобы внутренние
  проверки соединения не обходили туннель после выбора приложений.
- При сохранении правил активный VPN безопасно перезапускается и применяет
  новый allowed/disallowed-list без ручного отключения.
